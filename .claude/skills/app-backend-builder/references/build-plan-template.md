## Notes for the executing agent

- Read `convex-react-client.md` and `convex-auth-setup.md` in this kit before starting.
- Run `npx convex dev` in a separate terminal — it auto-pushes schema changes and gives you instant TypeScript types under `convex/_generated/`.
- After each phase, commit with a clear message: `git commit -m "Phase N: <what>"`.
- Don't skip self-check between phases. Bugs caught late cost much more than bugs caught at phase boundary.
- If you find existing Convex code that conflicts with the plan (e.g. a `tasks.ts` already exists with a different schema), STOP and ask the user — don't overwrite blindly.
