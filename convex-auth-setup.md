# Convex Auth — Setup, Patterns (Next.js / Web)

> **When to use:** AUTH = Convex Auth in `CLAUDE.md`. This is the official Convex auth library — `@convex-dev/auth`. Use this when the app shares a Convex DB with an existing RN app (so RN + web use the same identity layer), or when you specifically don't want Clerk.
>
> **Source:** Convex official docs — https://labs.convex.dev/auth and https://docs.convex.dev/auth. Re-fetch from the labs URL if patterns drift; Convex Auth is still in active development.

---

## What this is

`@convex-dev/auth` is auth that lives inside the Convex deployment. One library handles:

- **Email/password** — standard sign-up/sign-in flow (most common for B2B SaaS)
- **OAuth** — Apple, Google, GitHub providers
- **Magic links / OTP** — passwordless
- **Anonymous users** — every device gets a stable user record (rare for web; common for RN)

The `userId` lives in `ctx.auth.getUserIdentity()` — same shape Clerk uses, so server-side code is nearly identical to a Clerk-based app.

**Why this over Clerk:**
- One service, one dashboard (Convex), one set of env vars
- Same library works on RN and Next.js when both share a Convex DB — single source of truth for users
- No third-party identity vendor to worry about

**Why Clerk over this:**
- Polished prebuilt UI components (sign-in pages, user button)
- Better-supported in Next.js community
- Built-in org/multi-tenant features

The kit defaults to Clerk for new SaaS apps. Use Convex Auth when the cross-platform-DB sharing reason applies.

## Install

```bash
npm install @convex-dev/auth @auth/core@0.37.0
```

Or run `npx @convex-dev/auth` from the project root to scaffold all the boilerplate (`auth.ts`, `http.ts`, `auth.config.ts`, schema additions) automatically. This is the recommended path — match by hand only if you have a reason to.

You'll need the latest Convex client too — check `package.json` for `convex` and ensure it's recent.

## Server-side setup

Convex Auth splits into three files. Knowing which file owns what saves debugging time later.

### 0. `convex/auth.config.ts` (auto-generated)

This is a separate JWT-validation config file, generated by `npx @convex-dev/auth` (or copy from https://labs.convex.dev/auth/setup/manual). It exports a default config of shape `{ providers: [{ domain: process.env.CONVEX_SITE_URL, applicationID: "convex" }] }`. You generally don't edit it after setup — it's used by Convex to validate the JWTs the auth library issues.

### 1. `convex/auth.ts`

This declares what providers are enabled and exports the auth handlers.

```ts
import { Password } from '@convex-dev/auth/providers/Password';
import { convexAuth } from '@convex-dev/auth/server';

// For OAuth, add:
// import Apple from '@auth/core/providers/apple';
// import Google from '@auth/core/providers/google';

// For passwordless / anonymous, add:
// import { Anonymous } from '@convex-dev/auth/providers/Anonymous';

export const { auth, signIn, signOut, store, isAuthenticated } = convexAuth({
  providers: [
    Password,
    // Google,
    // Apple,
  ],
});
```

Do not import `Anonymous` from `@auth/core` — that's the Auth.js core lib. Convex Auth's wrapped provider lives at `@convex-dev/auth/providers/Anonymous` (capital `A` in path).

### 2. `convex/http.ts`

Wire the auth HTTP routes so the client can hit them.

```ts
import { httpRouter } from 'convex/server';
import { auth } from './auth';

const http = httpRouter();
auth.addHttpRoutes(http);
export default http;
```

### 3. `convex/schema.ts`

The auth library expects a `users` table. The `npx @convex-dev/auth` scaffold patches your schema for you. If you're hand-writing it:

```ts
import { defineSchema, defineTable } from 'convex/server';
import { authTables } from '@convex-dev/auth/server';
import { v } from 'convex/values';

export default defineSchema({
  ...authTables,
  // your own tables go here, keyed by userId from ctx.auth
  todos: defineTable({
    userId: v.id('users'),
    text: v.string(),
    done: v.boolean(),
  }).index('by_user', ['userId']),
});
```

## Client-side setup (Next.js App Router)

### 1. Provider wrapper

Create `app/ConvexClientProvider.tsx`:

```tsx
"use client";

import { ReactNode } from "react";
import { ConvexReactClient } from "convex/react";
import { ConvexAuthNextjsProvider } from "@convex-dev/auth/nextjs";

const convex = new ConvexReactClient(
  process.env.NEXT_PUBLIC_CONVEX_URL!
);

export default function ConvexClientProvider({ children }: { children: ReactNode }) {
  return (
    <ConvexAuthNextjsProvider client={convex}>
      {children}
    </ConvexAuthNextjsProvider>
  );
}
```

Wrap the root layout:

```tsx
// app/layout.tsx
import ConvexClientProvider from "./ConvexClientProvider";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <ConvexClientProvider>{children}</ConvexClientProvider>
      </body>
    </html>
  );
}
```

### 2. Middleware (App Router server-side auth)

Create `middleware.ts` at repo root:

```ts
import { convexAuthNextjsMiddleware, createRouteMatcher } from "@convex-dev/auth/nextjs/server";

const isPublicRoute = createRouteMatcher([
  "/",
  "/sign-in",
  "/sign-up",
  "/privacy",
  "/terms",
]);

export default convexAuthNextjsMiddleware((request, { convexAuth }) => {
  if (!isPublicRoute(request) && !convexAuth.isAuthenticated()) {
    return convexAuth.redirectToSignIn();
  }
});

export const config = {
  matcher: ["/((?!.+\\.[\\w]+$|_next).*)", "/", "/(api|trpc)(.*)"],
};
```

### 3. Sign-in / sign-up pages

Build at `app/sign-in/page.tsx` and `app/sign-up/page.tsx`. These are unstyled by default — you build the UI consuming `useAuthActions` from `@convex-dev/auth/react`:

```tsx
"use client";

import { useAuthActions } from "@convex-dev/auth/react";
import { useState } from "react";
import { useRouter } from "next/navigation";

export default function SignInPage() {
  const { signIn } = useAuthActions();
  const router = useRouter();
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");

  return (
    <form
      onSubmit={async (e) => {
        e.preventDefault();
        await signIn("password", { email, password, flow: "signIn" });
        router.push("/");
      }}
    >
      <input type="email" value={email} onChange={(e) => setEmail(e.target.value)} required />
      <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} required />
      <button type="submit">Sign in</button>
    </form>
  );
}
```

Sign-up is the same, with `flow: "signUp"`.

### 4. Sign-out button

```tsx
"use client";
import { useAuthActions } from "@convex-dev/auth/react";

export function SignOutButton() {
  const { signOut } = useAuthActions();
  return <button onClick={() => signOut()}>Sign out</button>;
}
```

### 5. Reading auth state in components

```tsx
"use client";
import { useConvexAuth } from "convex/react";

export function AuthGate({ children }: { children: React.ReactNode }) {
  const { isAuthenticated, isLoading } = useConvexAuth();
  if (isLoading) return <p>Loading…</p>;
  if (!isAuthenticated) return <a href="/sign-in">Sign in</a>;
  return <>{children}</>;
}
```

## Reading the user in Convex functions (server-side)

```ts
// convex/todos.ts
import { query, mutation } from "./_generated/server";
import { getAuthUserId } from "@convex-dev/auth/server";
import { v } from "convex/values";

export const list = query({
  args: {},
  handler: async (ctx) => {
    const userId = await getAuthUserId(ctx);
    if (!userId) return [];
    return await ctx.db
      .query("todos")
      .withIndex("by_user", (q) => q.eq("userId", userId))
      .collect();
  },
});

export const add = mutation({
  args: { text: v.string() },
  handler: async (ctx, { text }) => {
    const userId = await getAuthUserId(ctx);
    if (!userId) throw new Error("Not authenticated");
    return await ctx.db.insert("todos", { userId, text, done: false });
  },
});
```

Every query/mutation that touches user data calls `getAuthUserId(ctx)` and filters by it. Apps that skip this step leak data between users.

## Password reset (requires Resend)

If using the Password provider, password reset emails need an email sender. Convex Auth integrates with Resend:

1. Install Resend: `npm install resend` (also done at Stage 9 if EMAIL = Resend)
2. Configure the Password provider with Resend in `convex/auth.ts`:
   ```ts
   import { Password } from '@convex-dev/auth/providers/Password';
   import { Resend } from '@convex-dev/auth/providers/Resend';
   
   export const { auth, signIn, signOut } = convexAuth({
     providers: [Password({ verify: Resend })],
   });
   ```
3. Set `RESEND_API_KEY` and `AUTH_RESEND_KEY` in Convex deployment env vars (via Convex dashboard or `npx convex env set`)

Read https://labs.convex.dev/auth/config/passwords#email-verification for the full pattern.

## Production env vars

Set in Vercel project settings (and Convex deployment env vars where noted):

| Var | Where | Notes |
|-----|-------|-------|
| `NEXT_PUBLIC_CONVEX_URL` | Vercel | The Convex deployment URL |
| `CONVEX_SITE_URL` | Convex env | Set to `https://yourdomain.com` after Stage 12 |
| `RESEND_API_KEY` | Convex env | If using Password provider with reset |
| OAuth client IDs/secrets | Convex env | Per OAuth provider configured |

## Common pitfalls

| Pitfall | Fix |
|---------|-----|
| Forgot to wrap layout in `<ConvexAuthNextjsProvider>` | Auth state shows `undefined` everywhere. Add the wrapper. |
| Middleware not matching protected routes | Check the `matcher` config and `isPublicRoute` list. |
| Queries return empty for logged-in users | Forgot `getAuthUserId(ctx)` in the query handler. |
| Password reset email never arrives | Resend not configured in Convex env vars OR Resend domain not verified. |
| OAuth redirect fails in production | `CONVEX_SITE_URL` not updated to production domain. |

## Account deletion (B2B SaaS GDPR / settings page)

Most B2B SaaS apps need a self-serve "delete my account" button. Pattern:

```ts
// convex/users.ts
import { mutation } from "./_generated/server";
import { getAuthUserId } from "@convex-dev/auth/server";

export const deleteSelf = mutation({
  args: {},
  handler: async (ctx) => {
    const userId = await getAuthUserId(ctx);
    if (!userId) throw new Error("Not authenticated");
    
    // Delete user-owned records first (cascade)
    const todos = await ctx.db
      .query("todos")
      .withIndex("by_user", (q) => q.eq("userId", userId))
      .collect();
    for (const todo of todos) await ctx.db.delete(todo._id);
    
    // Then delete the user record
    await ctx.db.delete(userId);
  },
});
```

Wire from a settings page with a confirmation dialog. After successful delete, call `signOut()` and redirect to `/`.
